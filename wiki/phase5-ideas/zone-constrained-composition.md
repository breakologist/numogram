# Zone-Constrained Composition — Closed‑Loop Control Interface

**Phase:** 5 — Closed‑Loop Hyperstition  
**Milestone:** M1 — Control Interface (Zone-Constrained Composition)  
**Council consensus (2026‑05‑02):** Highest priority; feasibility confirmed.

---

## TL;DR

The `ModComposer` gains explicit zone orientation:

```python
composer = ModComposer(title="Zone2Tribute")
composer.target_zone(zone=2, brightness="warm")
composer.add_section(length=32, channel=0)
composer.write_mod("output.mod")
```

Generated tracks must classify as the target zone ≥90% of the time (Fifth Current empirical gate).

---

## Problem

Phases 1–4 proved that a robust classifier can distinguish zones in synthetic data (97.78% top‑1) and that real audio clusters only in zones 1, 2, 7. The absence of zones 3‑5, 8‑9 in wild recordings is a *real hyperstitional gap*, not an artifact.

We need a composer that can **deliberately** emit any zone, closing the loop:

> **Target zone → Generate → Classify → Verify.**

---

## Solution: `mod-writer-composer` Skill

A new Hermes skill (`numogram-audio/mod-writer-composer`) extends `ModComposer` with zone‑aware composition.

### Skill Location

- **Path:** `~/.hermes/skills/numogram-audio/mod-writer-composer/`

### API

| Method | Purpose |
|--------|---------|
| `target_zone(zone, brightness=None, density=None, gate_bias=None, bpm_range=None, waveform=None)` | Configure zone target and acoustic parameters |
| `add_section(length, channel=0, **overrides)` | Generate a pattern using current zone parameters |
| `compute_zone_centroids()` | Load/compute per‑zone MIR statistics from synthetic corpus |

**Parameter details:**

| Parameter | Values | Effect |
|-----------|--------|--------|
| `zone` | 1‑9 | Pentatonic degree + waveform/gate affinity |
| `brightness` | `"dark"`, `"warm"` (default), `"bright"` | Scales spectral centroid target (×0.7, ×1.0, ×1.5 vs zone median) |
| `density` | float 0‑1 | Note‑cell occupancy probability |
| `gate_bias` | `"none"`, `"arpeggio"`, `"slide"`, `"volume"`, `"syzygy"` | Weights gate family selection toward that effect family |
| `bpm_range` | `(min, max)` | Beats‑per‑minute window; defaults zone‑specific from corpus medians |
| `waveform` | `"square"`, `"triangle"`, `"noise"`, `"adaptive"` | Channel instrument sample mapping |

### Zone Parameters Source

1. **Computed centroids** (preferred) — `zone_centroids.json` generated by `scripts/compute_zone_centroids.py` from the synthetic 900‑track corpus (`synthetic_900_balanced.jsonl`). Captures centroid mean/std, BPM, density per zone.
2. **Fallback hard‑coded** (`ZONE_DEFAULTS`) — gentle hand‑tuned approximations matching pentatonic psychology.

## Validation Protocol (Fifth Current)

Every M1 deliverable must answer: *how will we know it works?*

```
P(success) ≥ 0.90  (zone classification confidence on generated tracks)
```

Procedure:
1. Generate *N* patterns with `target_zone(N)` fixed (default density 0.3, brightness "warm")
2. Classify each `.mod` file with `mod-writer-classifier`
3. Compute hit rate: `fraction where predicted_zone == target_zone`
4. If rate < 0.90, adjust `ZONE_DEFAULTS` or centroid calibration, repeat

Baseline (synthetic 900‑track classifier): 97.78% (train/test). Real‑world bias: expect ≥90% on targeted generation.

## Developer Notes

### Adding New Zone Parameters

Edit `ZONE_DEFAULTS` in `composer_extension.py` and/or regenerate `zone_centroids.json` via:

```bash
python ~/.hermes/skills/numogram-audio/mod-writer-composer/scripts/compute_zone_centroids.py
```

### Running Validation

```bash
python ~/.hermes/skills/numogram-audio/mod-writer-composer/scripts/validate_zone_bias.py --zone 2 --rounds 50 --length 16 --outdir /tmp/zone2_validation
```

Expected output line: `Target zone: 2  →  Hit rate: 46/50 = 92.0%` (≥90%).

### Dependency Warning

The skill imports `mod_writer.mapping.note_and_octave_from_zone`. Ensure `hermes.skills.numogram_audio.mod_writer` is importable (install the `mod-writer` skill first). The package uses relative imports within the `mod_writer` package.
