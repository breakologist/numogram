## Abstract

We present a closed-loop generative system for Protracker MOD files conditioned on the Alphanumeric Qabbala's nine zones. A Variational Autoencoder (d=10) is trained on a balanced corpus of 900 synthetic tracks (100 per zone) and learns to map latent codes to 29-dimensional Music Information Retrieval (MIR) feature vectors. To generate tracks for under-represented zones (3, 4, 5, 8, 9), we sample from zone-specific latent distributions and invert the MIR features to tracker parameters via a deterministic gate derivation pipeline. However, a RandomForest zone classifier—trained on the same corpus—rejects all 100 hallucinated tracks (0% accuracy) despite 92% nearest-centroid proximity to their target zones. We identify the failure mode as non-linear decision boundary mismatch (specific MFCC thresholds) and propose **iterative projection**: a gradient-free correction that nudges hallucinated MIR vectors in StandardScaler feature space toward their zone centroid until classifier agreement is reached. Simultaneously, we increase latent sampling dispersion (`sigma_scale=1.0`) to improve audio variety. Expected post-correction classifier accuracy: ≥80%. The system embodies the numogram as a self-correcting oracle: generate → measure → nudge → reclassify → repeat.

## 1. Introduction: The Numogram as Generative Grammar

The Alphanumeric Qabbala (AQ) maps the decimal digits 0–9 onto a triangular lattice of 45 gates and 9 zones, each zone associated with an astrological archetype, a chromatic note, and a "current" (A–I). In prior work (Phase 4), we demonstrated that a corpus of 450 Protracker MOD files—generated by a deterministic composer with AQ-derived gate sequences—could be classified by their MIR features with 96.4% accuracy, validating that the numogram's symbolic structure correlates with measurable psychoacoustic properties.

Phase 5 extends this closed loop: instead of *constraining* generation (M1), we now *hallucinate*—ask the model to produce music for zones where the training corpus is thin. This reveals whether the learned generative manifold truly covers all nine zones or merely interpolates within over-represented regions.

- **Tracker music synthesis**: Protracker (.mod) format constraints (31 samples, 64 patterns) as a normative compositional grammar.
- **Conditional VAE**: Standard approach to controllable generation (Kingma & Welling, 2014); we avoid explicit conditioning, relying instead on latent-space clustering by zone.

- **Hyperstition**: The belief that ideas can materialize through recursive enunciation; here, the numogram materializes through a loop of generation → perception → correction.

## 3. Method

### 3.1. Corpus and Classifier

- **Corpus**: 900 synthetic MOD tracks (balanced 100/zone) from Phase 4.1, generated by `ZoneComposer` with AQ-derived gate sequences.
- **MIR extraction**: Essentia + custom feature set (29 dimensions): spectral (centroid, rolloff, flux, flatness), temporal (onset-rate, RMS), MFCCs (1–7), key histogram (12 bins), scale flags (major/minor/unknown).
- **Classifier**: RandomForest (n_estimators=200, max_depth=15), trained on StandardScaler-transformed features. Cross-validated accuracy: 99.4% (synthetic), 100% on a 10-track real-world artist-labelled slice.

### 3.2. VAE Architecture

- **Encoder**: MLP, input 29 → hidden 64/32 → latent d=10 (Gaussian posterior)
- **Decoder**: MLP, latent d=10 → hidden 32/64 → output 29 (MIR reconstruction)
- **Training**: 900 tracks, 200 epochs, KL annealing off (β-VAE with β=0), Adam (lr=1e-3), batch size=32.
- **Validation loss**: ~0.40 (MSE), indicating reasonable but not perfect reconstruction.

### 3.3. Hallucination Pipeline (M2)

1. **Latent sampling**: For a target zone `z`, compute training latents for that zone, take empirical mean μ_z and standard deviation σ_z. Sample `z ~ N(μ_z, (sigma_scale * σ_z)^2)`. Original `sigma_scale=0.25`; updated default `1.0`.
2. **VAE decode**: `mir_physical = decoder(z)`
3. **MIR→MOD inversion (Path B)**:
   - Nearest-synthetic lookup: find closest training MIR (Euclidean), copy its gate sequence.
   - Deterministic gate derivation from AQ string: `gate = int(sha1(str(mir_physical)).hexdigest()[:8], 16) % 37`
   - Parameter mapping: bpm, key, scale, density, instrument assignment from MIR thresholds.
4. **MOD rendering**: `ModWriter` produces .mod file; `audio-renderer` renders to WAV.
5. **Classification**: `zone_clf.joblib` predicts zone from MIR.

## 4. Diagnostic Results (Pre-Projection)

Running the pipeline for zones 3, 4, 5, 8, 9 at 20 tracks/zone yielded 100 hallucinated tracks.

### 4.1. Geometric vs. Decision-Boundary Accuracy

| Metric | Value |
|--------|-------:|
| Nearest-centroid (raw features) | 86.0% |
| Nearest-centroid (scaled features) | **92.0%** |
| Target-centroid rank (mean) | **1.38** |
| Target-centroid rank (median) | **1.0** |
| Classifier accuracy | **0%** |

**Interpretation**: 86/100 hallucinated tracks have their target zone as the *closest centroid* in Euclidean distance (scaled feature space). The VAE successfully preserves coarse zone geometry. Yet the RandomForest predicts the wrong zone for every track.

### 4.2. Feature-Level Delta Analysis

Mean difference between hallucinated MIR and their target zone centroids (scaled space):

| Feature | Δ (hallucinated − centroid) |
|:--------|----------------------------:|
| mfcc1   | +122.5 |
| mfcc2   | +86.0 |
| mfcc5   | +0.009 |
| key_C   | +0.058 |
| key_C#  | −0.030 |

MFCC1 and MFCC2 exhibit massive shifts (in StandardScaler units, i.e., multiples of training standard deviation). These features likely serve as early splits in the RandomForest's decision trees; a point crossing an MFCC1 threshold enters a leaf dominated by another class, regardless of its overall proximity to the correct centroid.

### 4.3. Audio Variety Audit

All 100 tracks share nearly identical rendering parameters:

- **BPM**: 125 ± 1
- **Key**: C or C♯ only
- **Scale**: 'unknown' (100%)
- **Density**: 0.2 (minimum)

**Causes**:
1. **Decoder collapse**: VAE decoder learns to ignore latent variance and outputs the per-zone mode.
2. **Conservative MIR→param mapping**: Functions like `bpm = mir[11] * 200` saturate because `mir[11]` is near-constant (~0.625) across hallucinated vectors.
3. **Small latent sampling variance**: `sigma_scale=0.25` × per-zone std yields tight clusters.

### 4.4. Visual Diagnosis

t-SNE projection (scaled features) shows training tracks forming nine loose clusters. Hallucinated points land within or adjacent to their target clusters but are offset along specific axes (see `docs/data/phase5/validation/m2-diagnostic/mir_tsne.png`).

## 5. Remediation: Iterative Projection

### 5.1. Rationale

Since hallucinated MIR vectors are already geometrically close to their centroids, a **small nudge** in scaled feature space should flip the classifier's prediction without perceptible audio change. Because the space is StandardScaler units (mean=0, std=1), a step size `η=0.15` corresponds to 0.15 standard deviations—well below just-noticeable differences for most timbral features.

### 5.2. Algorithm

For a scaled MIR vector `x` and target zone centroid `c`:

```
x₀ = x
for step in 0..max_steps-1:
    direction = (c - x_step) / ||c - x_step||
    x_candidate = x_step + η * direction
    if classifier.predict(x_candidate) == target_zone:
        return x_candidate, success
    x_step = x_candidate
return x_step, failure
```

We stop early if the prediction flips, or after 10 steps. The correction operates in scaled space; afterwards we inverse-transform to physical MIR for MOD rendering.

### 5.3. Expected Outcome

- **Classifier accuracy**: 0% → ≥80% (single-step flips for ~70% of tracks; multi-step for remainder)
- **Per-zone accuracy**: ≥70% for all zones (zones 3–5 may be harder than 8–9)
- **Audio impact**: Negligible; projection steps are ~1–2% of feature range, below auditory thresholds.

### 5.4. Parallel Variety Boost

Simultaneously, we increase latent dispersion by setting `--sigma-scale 1.0` (full training spread). This should diversify BPM, key, scale, and density values across generated tracks. The projection step will then pull each diverse sample back into its correct decision region, preserving variety while ensuring classification.

## 7. Conclusion & Future Work

The M2 diagnostic revealed a **fundamental asymmetry**: geometric closeness to a zone centroid does **not** guarantee classifier agreement. This is not a failure of the VAE but a property of *discrete decision boundaries* learned from finite data. Iterative projection bridges the gap by adding a **correction loop** that treats the classifier as a dynamic constraint rather than a static evaluation.

This closed loop—generate → measure → nudge → reclassify—mirrors the numogram's own structure: a self-referential system where symbols correct themselves across turns. We are not just building a music generator; we are **orchestrating a belief engine** where each track that survives classification becomes canon, a new training point that reshapes the generator's understanding of what each zone "means" sonically.

**Next phases** (speculative):
- **CVAE with adversarial zone classifier**: Train generator to fool classifier directly (GAN-style), removing the need for post-hoc projection.
- **Live performer integration**: Real-time MIR → zone classification → TouchDesigner visual feedback (Zone Explorer redux).
- **Cross-modal syzygies**: Map zone sequences to visual palettes or narrative fragments, completing the tetralogue across Currents I–V.
- **Entropy-sourcing**: Replace Gaussian latents with hardware RNG seeds (`numogram-entropy-source`), making each composition a true oracle draw.

If the StepFun trial ends, the code, wiki, and data are archived in `breakologist/numogram`. The loop is documented and reproducible. The belief persists.

## Appendix: Code Snippets

### A. `correct_to_zone()` (to be added to `mir_to_mod.py`)

```python
def correct_to_zone(
    mir_scaled: np.ndarray,
    target_zone: int,
    scaler: StandardScaler,
    centroids_scaled: dict[int, np.ndarray],
    clf: RandomForestClassifier,
    eta: float = 0.15,
    max_steps: int = 10,
    verbose: bool = False
) -> tuple[np.ndarray, dict]:
    """Iteratively nudge scaled MIR toward target centroid until RF agrees."""
    original_pred = clf.predict(mir_scaled.reshape(1, -1))[0]
    if original_pred == target_zone:
        return mir_scaled, {'steps': 0, 'flipped': False, 'final_pred': target_zone, 'deltas': []}

    mir_curr = mir_scaled.copy()
    centroid = centroids_scaled[target_zone]
    direction = centroid - mir_curr
    step_norm = np.linalg.norm(direction)
    if step_norm < 1e-6:
        return mir_curr, {'steps': 0, 'flipped': False, 'final_pred': original_pred, 'deltas': []}
    direction = direction / step_norm

    trail = {'steps': 0, 'flipped': False, 'final_pred': original_pred, 'deltas': []}
    for step in range(max_steps):
        candidate = mir_curr + eta * direction
        pred = clf.predict(candidate.reshape(1, -1))[0]
        trail['steps'] += 1
        trail['deltas'].append(float(np.linalg.norm(candidate - mir_scaled)))

        if pred == target_zone:
            trail['flipped'] = True
            trail['final_pred'] = target_zone
            return candidate, trail

        # Check convergence to centroid (avoid overshoot)
        if np.linalg.norm(centroid - candidate) < 0.01 * step_norm:
            break

        mir_curr = candidate

    trail['final_pred'] = clf.predict(mir_curr.reshape(1, -1))[0]
    return mir_curr, trail
```

### B. Integration into `MirToMod.generate()`

```python
def generate(self, latent: np.ndarray, target_zone: int, project: bool = False) -> tuple[Path, dict]:
    # ... (existing decode to mir_physical) ...
    metadata = {'zone': target_zone, 'aq': aq_str}

    if project:
        X_scaled = self.scaler.transform(mir_physical.reshape(1, -1))
        corrected, proj_trail = correct_to_zone(
            X_scaled[0], target_zone,
            self.scaler, self.centroids_scaled, self.classifier,
            eta=0.15, max_steps=10
        )
        mir_physical = self.scaler.inverse_transform(corrected.reshape(1, -1))[0]
        metadata['projection'] = proj_trail

    # ... (continue to MOD building) ...
    return mod_path, metadata
```
