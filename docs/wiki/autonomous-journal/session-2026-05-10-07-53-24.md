## 2. Exploration: Root Cause Analysis

### Investigation Approach
I conducted a systematic diagnosis of the MIR feature extraction pipeline:

1. **Generated test audio** for pure Zone 1 using the `mod-writer` toolchain
2. **Measured spectral centroid** via MIRFeatureExtractor and direct FFT analysis
3. **Compared** against the synthetic training dataset (dataset_balanced_900.npz)

### Key Measurements
- **Our render**: centroid ≈ 324 Hz, bandwidth ≈ 761 Hz, sample rate 48000 Hz
- **Training dataset**: centroid range 4817-9683 Hz, mean 6923 Hz

### Discovery: Sample Rate Mismatch
The synthetic dataset was generated with **higher-quality audio** (44.1/48 kHz samples), while the current pipeline uses **8363 Hz square wave samples**. This causes:
- Band-limited spectrum (Nyquist ~4 kHz)
- Low spectral centroid
- Classification bias

### Waveform Analysis
- Generated samples are very short (0.15 sec) and use only ±40 amplitude (low energy)
- Square wave harmonics are weak; most energy concentrated below 540 Hz
- The MOD format transposes samples, further lowering fundamental frequency

### Attempted Fixes
- Increased sample duration to 1 second → no significant change
- Changed sample rate to 44100 Hz → no significant change
- Switched to brighter waveforms (sawtooth) → marginal improvement
