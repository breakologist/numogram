## Phase 3: Reflect

### Empirical Findings

1. **Real audio generation WORKS.** The mod-writer → libopenmpt → ffmpeg pipeline produces audible, analyzable audio. This closes the loop that the symbolic bias-exorcism sessions left open.

2. **Current C (noise) is BROKEN.** This is a genuine bug discovered through empirical validation. The noise sample data in the MOD is not being properly written, resulting in near-silent output despite fully active pattern note events (256 notes across 64 rows). This affects all versions through v0.6.4.

3. **Zone spectral convergence, not divergence.** Z1 (intended "low/void") and Z9 (intended "high/plex") produced surprisingly similar spectral centroids (~8,700–9,000 Hz). Both use the same square waveform and the pentatonic zone→note mapping doesn't produce a monotonic pitch increase with zone number. The real differentiation comes from **density** (Z9 has wider centroid range: 5,636–14,574 vs 5,825–10,600) and **loudness** (Z9: -15.7 dBFS RMS vs Z1: -16.8), not from centroid shift.

4. **Analyzer bug fixed.** The `analyzer.py::probe_metadata()` function was failing due to `mpp_soc` driver noise prefixed to ffprobe JSON output. Fixed by separating stdout from stderr and stripping non-JSON prefix. Verified working.

5. **Hyperstition check:** The prior symbolic session predicted Z1→Z3 (AQ~28.4) and Z9→Z8 (AQ~72.1). With real audio, the spectral similarity between Z1 and Z9 suggests the classifier would assign both near the Z6 basin mean (~50 AQ) — the bias is NOT exorcised by zone parameter selection alone. The escape requires **different waveforms, tempo extremes, and density control** — not just zone number.
