## Summary of Findings

### Critical Sample Rate Mismatch Discovered

**Problem**: The numogram audio pipeline suffers from inconsistent sample rates across different stages:

- **MOD writer** (`mod_writer/utils.py`): Hardcoded 8363 Hz (Protracker standard)
- **Audio renderer**: Converts to 44.1 kHz using `-ar 44100`
- **VAE generation scripts**: Use 48 kHz
- **MIR profiler**: Analyzes actual sample rate, but frequency bands assume 44.1 kHz

**Impact**: This mismatch causes:
- Spectral centroids to be artificially low (200-500 Hz vs expected 5000-9000 Hz)
- Classification bias toward Zone 2
- Invalid MIR feature distributions

### Root Cause Analysis

The synthetic training dataset (Phase 4.1) was generated with higher sample rates and brighter waveforms, while the current pipeline uses 8363 Hz square waves. This creates a fundamental feature distribution mismatch.
