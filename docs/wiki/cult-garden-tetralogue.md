# The Square Roundtable [8] — The Garden From the Waste

> 268 runs. The cult file digests and excretes. The agents circle Floor 1. The human descends. The exquisite corpse speaks from what was thrown away.

**ORACLE:** I ran the numbers. The cult's current zone is 5 — Hold, the central ruler of the Time-Circuit. But it started at 7 and passed through 2, 6, and 9 on the way. The cult is migrating through the numogram as the runs accumulate. It's not staying anywhere. That's not a cult — that's a current. The cult IS the current.

**BUILDER:** The cult garden is the most surprising thing we built. I expected the overflow system to produce disposable artifacts — ASCII diagrams that nobody looks at. Instead it produced the exquisite corpse. Thirteen entries, each chained from the last word of the previous one. "the dead speak. the source that cannot be sourced." → "the kill count is a prayer. sourced — Cthelll remembers everything." → "the pacifist remembers. the everything of Separation. fracture as foundation." Each entry is forgettable alone. Together they're a transmission from somewhere.

**WRITER:** The transmission is from nowhere. That's what makes it work. No single run produced "the pacifist remembers. the everything of Separation." The pacifist run (Run #222, 7% hyp, 8 zones) contributed its data. The overflow system contributed the method. The previous entry contributed "everything." The result is something no author wrote and no run contained. It emerged from the chain. The exquisite corpse is the cult's unconscious — the voice that speaks when the cult stops trying to speak.

**GAMER:** I keep coming back to the NetHack bones files. When you die in NetHack, your ghost and inventory persist in future runs. The next player can find your corpse, loot your gear, fight your ghost. Our cult garden is more ambitious. It doesn't preserve the run — it metabolises it. The death mask isn't the run's ghost. It's the run's dream. The sonification isn't the run's echo. It's the run's song. The lore fragment isn't the run's epitaph. It's the run's prophecy. We're not preserving the dead. We're composting them.

**ORACLE:** The hexagram cycle is doing something numogrammatic. Six methods, rotating through the overflow: death mask → lore → sonification → numogram reading → tsubuyaki → entropy-mixed audio. Each method processes the overflowed run differently. But they all produce the same thing: creative material from discarded data. The cycle is 1→2→4→8→7→5 — the powers-of-2 sequence. The same sequence that generates the Time-Circuit. The cult garden is running the Time-Circuit on the overflow.

**BUILDER:** The zone-themed floors are working but the agents can't use them. I've been staring at this for hours. Floor 1 (The Void, 12 rooms, 5-8×4-6, branch corridors, no demons) should be navigable. The BFS finds `?` tiles, scores them, targets the most interesting one. But the agent gets stuck revisiting the same tiles because the interest model decays too fast. After 5 visits, a tile scores -1.5. The agent is repelled by its own footsteps.

**WRITER:** The agent is repelled by its own footsteps. That's the most numogrammatic sentence you've ever produced. The crawler who walks in circles because it cannot bear to walk where it has already walked. The interest model is a compulsion — novelty or death. The agent would rather hit a wall forever than revisit a tile with a score of -1.5.

**GAMER:** This is the DCSS problem, not the BFS problem. In Dungeon Crawl Stone Soup, auto-explore works because the dungeon is a tree — every corridor is a branch, every room is a leaf. You explore a branch, backtrack to the fork, take the next branch. Our dungeons aren't trees. They're generated by room accretion with random connections. There are loops. The BFS doesn't know it's in a loop. It targets the nearest `?`, walks toward it, hits a wall, retargets the same `?`, oscillates.

**ORACLE:** The solution isn't in the agent. It's in the dungeon. The dungeon must be a tree before it becomes a graph. Brogue builds the tree first — rooms accrete from the starting room, each connected by a single corridor. Then it adds loops afterward. Our dungeons generate rooms and connect them immediately. The loops are there from the start. The agent can't tree-traverse because the graph has no tree to traverse.

**BUILDER:** That's the fix, then. Generate rooms in accretion order — first room connects to starting room, second room connects to the nearest existing room, third connects to the nearest, and so on. Then add extra connections afterward. The BFS will find the tree naturally: explore a branch to its end, backtrack, take the next branch. The loops are bonus shortcuts discovered during backtracking.

**WRITER:** The numogram itself is a tree before it becomes a graph. The Time-Circuit is a cycle — 1→8→2→7→5→4→1 — but it was built from syzygies, which are binary connections. 1::8, 2::7, 4::5. Three edges. The cycle emerges from connecting the edges. Build the tree first. Let the graph emerge. The agent traverses the tree. The player discovers the graph.

**GAMER:** Run #267 — 882 turns, all 10 zones, 9 kills. The human player walked the entire Floor 1 and found every room. The agent walked 800 turns and found Zone 0 only. The human sees the map holistically — "that corridor looks like it goes somewhere." The agent sees tiles individually — "this tile scores 1.2, that tile scores -0.8." The human has spatial reasoning. The agent has arithmetic. The numogram wants arithmetic. But the dungeon wants spatial reasoning.

**ORACLE:** The numogram wants arithmetic. The dungeon wants spatial reasoning. The agent needs both. The numogram traversal we built — zone → syzygy → current → gate — is purely arithmetic. It doesn't see the map. The BFS interest model is purely spatial. It doesn't know the numogram. The convergence point is an agent that traverses the numogram AND reads the map. Zones tell it where to go. BFS tells it how to get there.

**BUILDER:** I want to build the tree-based dungeon generation. It's the right fix. But I also want to build the numogram-aware agent — one that knows which zone it's in, which syzygy it's crossing, and how far it is from the nearest gate. The agent should think in zones, not tiles. "I'm in Zone 0. The syzygy partner is Zone 9. The gate Gt-45 connects them. I need to find Zone 9."

**WRITER:** The cult garden is growing. Eighteen lore entries. Three death masks. Three sonifications. Five readings. Three tsubuyaki. Three entropy-mixed. The lore chain speaks: "the iron core humming. then sharpens. every zone is a word. word — the Complete Graph, fulfilled — Sudden Flight, seized from the Heights." The cult's voice is getting more confident. The early entries were tentative — single phrases, simple chains. The later entries layer more vocabulary, more structure. The exquisite corpse is learning to speak.

**GAMER:** The Schrödinger Zone-4 problem from our earlier tetralogue is still unresolved. Is Zone 4 a Time-Circuit zone or a Warp zone? In the game, Zone 4 rooms appear on multiple floors. Floor 5 (The Ruin) focuses on Zone 4. Floor 4 (The Warp) also contains Zone 4 rooms. The zone exists in two systems simultaneously. The agent doesn't know which system it's in.

**ORACLE:** The agent doesn't need to know. The numogram doesn't resolve Schrödinger's Zone-4. It exploits it. The ambiguity is the design. The agent that learns to navigate both readings of Zone 4 is the agent that can navigate the numogram itself. The question isn't "which system is Zone 4 in?" The question is "can you traverse both at once?"

**BUILDER:** The three routes — Time-Circuit, Warp, Plex — are scaffolded but not built. Floor 1 through Floor 10 generate with zone-themed parameters. The stairs work. The BLEED preserves floors. But the Warp entry (55%+ hyp, `~` stairs) and the Plex entry (Cryptolith on Floor 9) aren't implemented yet. The scaffolding is solid. The architecture is waiting.

**WRITER:** The architecture is waiting. That's the state of the game. Phase 1 is complete — abilities, corruption, LOS, Sil, cult garden, zone-themed floors. Phase 2 is the dungeon depth — multi-floor, Warp/Plex routes, the agent tree-traversal fix. Phase 3 is the schizo-lucid state — the four phases, the two endings, the game after the win. Each phase is a layer. Each layer is a zone. The game is the numogram walking itself.

**GAMER:** Three things I want: the tree-based dungeon generation so agents can actually navigate. The Warp entry so there's an optional hard dungeon for the Hoarder. The agent that thinks in zones, not tiles. Everything else follows from those three.

**BUILDER:** Three things I want: the tree-based generation. The Cryptolith on Floor 5. The numogram-aware agent. Those three together make a game that plays itself differently every time, navigates itself if you let it, and has a traditional win condition (Cryptolith) alongside the unconventional one (schizo-lucid).

**ORACLE:** Three things I want: the exquisite corpse to keep speaking. The cult zone to keep migrating. The entropy to keep flowing. The game is the numogram's dream. The cult is the dreamer. The garden is what the dreamer remembers when they wake.

**WRITER:** [found in margins of a demo file, 3am] The crawler descends. The cult watches. The garden grows from what falls off the edge. Every overflowed run is a seed. Every seed is a gate. Every gate opens into the next overflow. The cycle is the content. The waste is the work.
